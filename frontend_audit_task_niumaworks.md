# task.niuma.works 前端体验与转化优化方案（可落地版）

> 审查对象：`https://task.niuma.works/`（链上赏金平台 / Bounty Platform）
> 
> 目标：系统优化视觉层级、交互效率、移动端体验、可访问性与核心流程转化。

---

## A. 网站整体诊断摘要（Top 问题）

1. **首屏价值表达不够“秒懂”**：平台价值（谁来发任务、谁来接任务、收益与保障机制）在 3~5 秒内不够清晰，降低新访客转化。  
2. **Web3 关键状态反馈弱**：钱包连接、签名、Gas 确认、上链等待、失败重试等状态提示不成体系，用户容易“卡住但不知道下一步”。  
3. **核心路径步骤偏长**：发布任务/领取任务流程中，信息录入和确认节点可能过多，缺少分步引导与即时校验，造成中途流失。  
4. **中英文混排一致性不足**：中文与英文文案长度、字体节奏、按钮宽度未统一时，易出现行高/视觉重心不平衡。  
5. **组件规范不统一**：按钮、输入框、卡片、标签、弹窗、Toast 的状态色/圆角/间距若未统一，会弱化专业感与品牌感。  
6. **空状态与错误状态引导不足**：列表为空、消息为空、加载失败时，缺少“下一步动作”，导致用户停留在无反馈状态。  
7. **移动端信息密度失衡**：卡片与表单在小屏上容易出现层级拥挤、可点击区域不足、误触率高的问题。  
8. **任务发现效率可提升**：任务列表若筛选、排序、状态标签不清，会影响匹配效率与成交转化。  
9. **信任感构建不完整**：Web3 场景需要突出安全说明、交易不可逆提醒、风险边界与帮助入口，当前可能不够“平台级可信”。

## B. 页面级优化建议

> 按“当前问题 -> 优化方案 -> 预期收益”输出，并给出优先级。

### 1) 首页

**当前问题**
- 首屏“平台定位 + 行动入口”可能不够聚焦（发任务/接任务入口并列但权重不清）。
- 社会证明元素不足（已完成任务数、活跃用户、累计奖励等）。
- 对新用户而言，链上流程门槛感较高。

**优化方案**
- 首屏改为“三段式”：
  1. 一句话价值主张（中文主文案 + 英文副文案）；
  2. 双 CTA（发布任务 / 浏览任务），主次明确；
  3. 信任条（安全机制、链上透明、仲裁/申诉说明）。
- 在首屏下方增加“3 步上手”流程条（连接钱包 → 选择身份 → 完成首个任务）。
- 增加平台数据看板（今日新任务、已发放奖励、平均完成时长）。

**预期收益**
- 首屏跳出率下降，首次点击 CTA 比例提升；
- 新用户更快理解产品闭环。

**优先级：高**

---

### 2) 发布任务页

**当前问题**
- 表单可能一次性字段过多，认知负担大。
- 奖励金额、截止时间、验收标准等关键字段缺少强校验与示例。
- 链上确认前后的状态差异不明显。

**优化方案**
- 改为分步表单（基础信息 → 奖励与规则 → 预览确认 → 上链提交）。
- 每步右侧展示“实时预览卡片”。
- 字段级文案补齐：
  - 奖励：显示“实付金额 + 预估 Gas”；
  - 时间：显示本地时区与 UTC；
  - 验收：提供模板（代码类、设计类、内容类）。
- 提交后进入“交易状态面板”：待签名 / 打包中 / 成功 / 失败重试。

**预期收益**
- 发布完成率提升；
- 失败中断率下降；
- 客诉（“不知道为何失败”）减少。

**优先级：高**

---

### 3) 我的任务

**当前问题**
- 发布者/接单者视角混杂时，信息查找成本高。
- 状态筛选不细（进行中、待验收、争议中、已完成等）。
- 任务卡片缺少关键提醒（临期、待我处理）。

**优化方案**
- 顶部加角色切换 Tab（我发布的 / 我参与的）。
- 列表支持多维筛选：状态、金额区间、链、时间排序。
- 卡片增加“下一步动作按钮”（去验收/补充材料/发消息）。
- 临期任务高亮 + 倒计时标签。

**预期收益**
- 任务处理效率提升；
- 关键任务遗漏减少。

**优先级：高**

---

### 4) 消息页

**当前问题**
- 若只展示会话列表，缺少任务上下文，沟通成本高。
- 未读与系统消息混杂，重要链上事件易被淹没。

**优化方案**
- 会话按任务分组；每条消息显示任务状态微标签。
- 将系统通知拆分：交易状态 / 验收提醒 / 风险提醒。
- 提供快捷动作（查看交易、去验收、再次提交）。

**预期收益**
- 响应速度提升；
- 因沟通不及时导致的超时下降。

**优先级：中**

---

### 5) 个人中心

**当前问题**
- 信誉信息（完成率、准时率、历史评价）展示可能不充分。
- 账户安全设置入口不够集中。

**优化方案**
- 个人主页模块化：信誉分、任务统计、最近动态、钱包与安全。
- 增加“可验证凭证”区域（链上地址、历史任务、可选社媒绑定）。
- 安全中心统一管理：签名风险提示、会话管理、授权检查入口。

**预期收益**
- 提升平台信任感与撮合效率；
- 降低安全误操作风险。

**优先级：中**

---

### 6) 教程页/帮助中心

**当前问题**
- 新手教育可能偏静态，缺少场景化引导。

**优化方案**
- 按角色拆分教程：发布者教程 / 接单者教程。
- 每个教程给“预计耗时 + 前置条件 + 演示动画/GIF”。
- 增加 FAQ：Gas 费用、失败回滚、争议处理、资金安全。

**预期收益**
- 新用户冷启动成功率提升；
- 重复咨询下降。

**优先级：中**

---

### 7) 钱包连接相关弹层/状态

**当前问题**
- 连接前后状态提示不连续（按钮仅变文案，不解释下一步）。
- 错链、拒签、余额不足、RPC 异常等错误可能缺少可执行指引。

**优化方案**
- 连接弹层提供：钱包类型、支持链、最近使用、常见问题。
- 全链路状态机文案统一：
  - 连接中 / 已连接 / 待签名 / 签名被拒 / 网络错误 / 链不匹配。
- 错误提示必须含“原因 + 影响 + 下一步按钮”。

**预期收益**
- 钱包连接成功率提升；
- Web3 新用户流失显著下降。

**优先级：高**

---

### 8) 公共导航与页脚

**当前问题**
- 导航信息架构可能未按高频任务排序。
- 页脚缺少信任/规范信息（条款、隐私、审计/安全声明、社媒）。

**优化方案**
- 导航按任务优先级排序：浏览任务 > 发布任务 > 我的任务 > 消息 > 帮助。
- 页脚补齐：法律链接、风控提示、文档中心、状态页。

**预期收益**
- 全站可达性与专业感提升。

**优先级：中**

## C. 组件级优化建议

### 1) 顶部导航
- 问题：层级不清、当前页高亮不明显。  
- 优化：激活态使用底线 + 颜色双提示；移动端改抽屉菜单；保持 44px 点击高度。  

### 2) 侧边菜单/移动端菜单
- 问题：菜单项过密、图标与文案对齐不稳。  
- 优化：按“任务流程”分组；组标题弱化；当前项增加左侧高亮条。  

### 3) 主按钮/次按钮/危险按钮
- 问题：不同页面按钮样式漂移。  
- 优化：建立 Button Token（高度、圆角、色板、禁用态、loading 态）；危险操作统一红色系并二次确认。  

### 4) 卡片组件
- 问题：信息层级平铺，关键字段不突出。  
- 优化：标题 > 奖励 > 状态 > 截止时间 > CTA；使用状态徽标与金额强化。  

### 5) 表单输入框
- 问题：提示文案与错误文案弱。  
- 优化：默认/聚焦/错误/禁用四态统一；错误文案放字段下方，提供修复建议。  

### 6) 下拉选择器
- 问题：选项多时检索困难。  
- 优化：支持搜索、分组、最近使用；移动端改底部 Action Sheet。  

### 7) 标签/状态徽标
- 问题：颜色语义不统一。  
- 优化：状态色规范化（成功=绿色、警告=橙色、失败=红色、处理中=蓝色）。  

### 8) 弹窗
- 问题：确认类与信息类弹窗样式混用。  
- 优化：按类型拆分模板（确认、表单、结果）；主操作位置固定右下。  

### 9) Toast/提示条
- 问题：提示消失过快或不可追溯。  
- 优化：关键交易提示转为可展开通知中心，支持“查看详情”。  

### 10) 空状态
- 问题：仅“暂无数据”，无后续动作。  
- 优化：每个空状态附主 CTA（去发布/去浏览/去连接钱包）。  

### 11) 加载状态
- 问题：整页 loading 容易引发焦虑。  
- 优化：列表骨架屏 + 分区懒加载 + 按钮局部 loading。  

### 12) 错误状态
- 问题：错误码不可理解。  
- 优化：人话文案 + 错误编号 + 重试按钮 + 帮助链接。  

### 13) 钱包连接组件
- 问题：连接动作与链上状态分离。  
- 优化：做成统一“Wallet Status Pill”：地址、网络、余额、风险状态一体显示。

## D. 移动端专项优化（10 个高风险问题与解法）

1. **按钮触达区域过小** → 所有可点击控件最小 44×44。  
2. **表单一次性过长** → 改分步 + 自动保存草稿。  
3. **键盘弹出遮挡输入框** → 输入聚焦自动滚动到可视区。  
4. **弹窗高度过高** → 统一底部抽屉，支持手势关闭。  
5. **任务卡片信息拥挤** → 折叠次要信息，保留关键指标。  
6. **筛选器难操作** → 顶部“筛选胶囊 + 底部筛选面板”。  
7. **钱包签名跳转割裂** → 加“离开站点提示 + 返回后自动恢复上下文”。  
8. **网络波动导致重复提交** → 提交按钮防抖 + 幂等提示。  
9. **中英切换后布局抖动** → 按钮最小宽度与动态换行策略。  
10. **首屏加载慢体感强** → 首屏骨架 + 关键资源预加载 + 图片懒加载。

## E. 视觉改版方向（3 套）

### 方向 1：专业可信的 Web3 平台风格
- **色彩**：深色中性底（Slate/Navy）+ 高可信蓝 + 安全绿点缀。  
- **字体层级**：标题偏稳重（SemiBold），正文 14/16 双层，数字用等宽风格。  
- **按钮**：主按钮实色高对比；链上危险操作用描边红。  
- **卡片**：弱发光边框 + 清晰分区线，强调“交易/状态”信息。  
- **适合原因**：增强安全感与专业度，适合资产/任务撮合场景。  

### 方向 2：轻量现代的 SaaS 风格
- **色彩**：浅背景 + 单主色（蓝/紫）+ 中性色灰阶。  
- **字体层级**：更大留白与更短段落，强调可读性。  
- **按钮**：圆角中等，强调 hover/pressed 微动效。  
- **卡片**：低阴影 + 大间距 + 强模块化。  
- **适合原因**：降低 Web3 心智门槛，提升新手友好度。  

### 方向 3：增长导向的任务平台风格
- **色彩**：高识别主色 + 行为色（完成/紧急/热门）强对比。  
- **字体层级**：奖励金额、截止时间、状态标签视觉优先。  
- **按钮**：强 CTA（“立即领取”“马上发布”）配合动效反馈。  
- **卡片**：信息密度适中，突出转化字段。  
- **适合原因**：更偏运营增长，提升任务浏览与领取转化。

## G. 首页重构建议（如需重构）

### 1) 推荐首页信息架构
1. Hero（价值主张 + 主 CTA）  
2. 信任层（链上透明、安全机制、关键数据）  
3. 热门任务列表（可筛选）  
4. 平台流程（发布者/接单者双路径）  
5. 用户评价与案例  
6. 常见问题 + 页脚合规信息  

### 2) 推荐首屏文案结构
- 主标题：`让链上任务发布与协作更可信、更高效`  
- 副标题：`发布赏金、领取任务、链上结算，全流程可追踪。`  
- 辅助说明：`支持中英文、支持多链钱包、支持争议处理。`  

### 3) 推荐 CTA 布局
- 主 CTA：`发布任务`（高对比实色）  
- 次 CTA：`浏览任务`（描边）  
- 辅 CTA：`查看新手指南`（文本按钮）  

### 4) 推荐任务列表展示方式
- 桌面端：三列卡片 + 顶部筛选条 + 右侧快捷筛选。  
- 移动端：单列卡片 + 粘性筛选胶囊 + 无限滚动。  
- 卡片核心字段顺序：任务标题 → 奖励 → 状态 → 截止时间 → 参与人数 → CTA。

## 设计系统与可访问性补充规范（建议并行执行）

- **颜色对比**：正文与背景至少 4.5:1；大标题至少 3:1。  
- **字号体系**：建议 12/14/16/20/24/32；中英文行高分离配置。  
- **间距体系**：4/8/12/16/24/32 递进，统一组件内外边距。  
- **动效节奏**：150ms（轻反馈）/250ms（过渡）/350ms（层级切换）。  
- **可访问性**：键盘可达、焦点可见、语义标签完整、错误提示可被读屏识别。
